I. Introduction

Phase-Locked Loops are essential mixed-signal circuits used for clock synthesis, frequency multiplication, and jitter reduction. The reference repository targets an 8× clock multiplier on the SKY130 open-source PDK: a 5 MHz reference input is multiplied to produce a 40 MHz locked output [1]. The entire flow — schematic capture, simulation, layout, and extraction — uses only free and open-source tools, making it an ideal learning platform.

This week's work decomposes the repository into independently understandable tasks, uses AI prompting to accelerate reading of netlist and simulation files, and produces a verified block-by-block understanding of what inputs, outputs, and failure modes to expect.

II. PLL Architecture Overview

The 8× PLL comprises five major sub-blocks connected in a negative-feedback loop (Fig. 1). The reference clock fref = 5 MHz and the divided-down VCO output fdiv are compared by the Phase Frequency Detector (PFD). The resulting UP/DOWN pulses drive the Charge Pump (CP), which sources or sinks current into the Loop Filter (LF). The resulting control voltage Vctrl tunes the Voltage-Controlled Oscillator (VCO), and a ÷8 Frequency Divider closes the feedback path [1].

fref →PFD →CP →LF →VCO →fout
↑               ÷8 Divider           ↓
Fig. 1. PLL feedback loop (8× multiplier)

The fundamental frequency relationship is:

fout = N · fref  →  40 MHz = 8 × 5 MHz

III. Sub-Block Analysis

A. Phase Frequency Detector

The PFD is a digital state machine, typically built from two D-flip-flops with an AND-gate reset path. When fref leads fdiv, it asserts UP; when fdiv leads, it asserts DOWN. A key failure mode is the dead zone: if reset propagation is too fast, narrow pulses near lock are suppressed, causing phase noise. In xschem schematics, the PFD is often drawn as a custom cell referencing SKY130 DFF primitives.

B. Charge Pump

The CP converts digital UP/DOWN pulses into an analog current. An ideal CP sources current ICP on UP and sinks the same on DOWN. Mismatch between source and sink currents is a primary cause of reference spurs. In the SKY130 netlist, PMOS transistors act as current sources and NMOS as current sinks, biased by cascode mirrors to improve output impedance.

C. Loop Filter

A passive second-order RC filter (R₁, C₁, C₂) converts CP current pulses to Vctrl. The zero introduced by R₁C₁ stabilises the loop. The loop bandwidth is approximately:

ωn = √(KVCO · ICP / (2π · N · C₁))

Stability requires phase margin > 45°. Choosing C₂ ≈ C₁/10 keeps the second pole well above ωn.

D. Voltage-Controlled Oscillator

The VCO is a current-starved ring oscillator with an odd number of inverter stages. Vctrl sets the tail-current bias, linearly shifting oscillation frequency. The VCO gain KVCO (MHz/V) is the slope of the f vs. Vctrl curve extracted from a sweep simulation. Ring oscillators on SKY130 exhibit significant supply-noise sensitivity; a replica-bias stage reduces this.

E. Divide-by-8 Frequency Divider

Three cascaded toggle flip-flops implement ÷2, ÷4, ÷8 division. The output of the third stage feeds fdiv back to the PFD. Layout placement of the divider close to the PFD minimises routing skew, which is critical for lock accuracy.

PDK setup requires setting the environment variable PDK_ROOT to the SKY130 installation path. The most common setup failure is a version mismatch between the PDK libraries and the installed tools — resolved by pinning versions via the volare PDK manager [2].

V. Pre-Layout vs. Post-Layout Simulation

Pre-layout simulations use ideal wire connections from the xschem netlist. They confirm circuit functionality but ignore parasitics. Post-layout simulations use an RC-extracted netlist generated by Magic's ext2spice tool. Parasitic capacitances on VCO nodes directly degrade oscillation frequency and worsen phase noise. Expected differences include: (a) VCO free-running frequency shifts by 5–20%; (b) lock time increases; (c) charge pump glitches may appear due to gate capacitance on switching nodes.

VI. Simulation Workflow & Debug Strategy

The recommended incremental simulation strategy is:

1.Verify VCO free-running range with a Vctrl DC sweep before any loop closure.
2.Test PFD UP/DOWN pulse widths with a two-clock transient stimulus.
3.Verify CP current balance by probing I(Vctrl) node directly.
4.Close the loop and run a full transient to observe lock; measure lock time as the interval from start to when |fout − 40 MHz| < 1%.
5.For post-layout, compare Vctrl settling waveform with pre-layout to quantify parasitic impact.
